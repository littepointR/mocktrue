# Security Policy

## Reporting a Vulnerability

Please do not publish sensitive vulnerability details in a public issue.

If GitHub private vulnerability reporting is available for this repository, use it first. Otherwise, open a GitHub issue with a short, non-sensitive summary and ask for a private follow-up channel.

Include the affected version or commit, the operating system, reproduction steps, expected behavior, observed behavior, and any relevant logs with secrets removed.

## Supported Versions

MockTrue is under active early development. Security fixes are applied to the default branch first.
