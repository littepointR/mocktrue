Multi-Agent Orchestration

脚本功能多 Agent 任务分配流程图

编排原则是先冻结接口，再并行实现。每个 agent 有单一职责，并且每个 agent 内部都按“需求确认、开发、审查、测试验证”的循环推进。

接口冻结后并行 单一职责 每轮可验证 阻塞问题回流修复
串行关口 0 冻结节点类型、端口、config、脚本 API、Monaco metadata。
并行开发 后端、前端、MCP/demo 分工开发，互不重叠写文件。
串行集成 合并接口、修复冲突，执行最终验证和提交。

每个 Agent 内部循环

无论 agent 负责哪一部分，都按同一套开发循环执行；差别只在写入范围和验证命令。

1. 需求确认 复述本 agent 的输入、输出、写入范围和验收标准。
2. 开发实现 先写失败测试，再做最小实现，避免扩大范围。
3. 审查验证 自查后运行 targeted tests，阻塞问题回流修复。

并行边界

并行的前提是共享接口被冻结；否则三边会各自定义字段，集成成本会很高。

可并行

  • 后端 runtime 和前端 Monaco。
  • MCP catalog 和 workspace tests 的准备部分。
  • 审查和 targeted verification。
  • demo 压测和最终 review。

必须串行

  • 节点类型、端口、config 和 API 冻结。
  • 共享 provider/schema 集成。
  • 修复跨模块字段不一致。
  • 最终完整验证和提交。
