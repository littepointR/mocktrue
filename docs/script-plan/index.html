PortWeave Script Nodes

把脚本能力变成安全可控的拓扑节点

脚本功能的目标不是让用户运行任意系统脚本，而是让用户在串口拓扑中编排可观察、可保存、可测试的数据处理逻辑。脚本只能使用 PortWeave 提供的 API，并通过节点端口或授权资源访问数据。

Monaco 编辑器 动态补全 受控 API 禁止任意系统访问
Generator 定时生成字节流，作为自动测试或模拟数据源
Transform 过滤、转换、追加校验、协议适配
Analyzer 字段标注、错误记录、帧历史展示
3 第一阶段脚本节点
1 第二阶段 endpoint
50ms 默认执行超时
0 停止后残留资源

功能阶段目标

这三个页面描述产品能力如何从最小闭环推进到资源访问和稳定化。每个页面都包含目标、实现、流程、步骤、验收和风险。

Phase 1：拓扑内脚本能力 实现 transform、generator、analyzer。脚本只通过连接线收发数据，配套 Monaco 编辑器和 API 提示。 Phase 2：受控串口 endpoint 脚本可以访问串口，但只能访问节点启动时授权的资源句柄，统一走资源冲突检查和清理。 Phase 3：增强和稳定化 补模板、日志、压力测试、长稳测试、API 细化和 Monaco 体验优化。

开发流程目标

这些页面描述多 agent 开发流程。每个目标都有单一职责，便于把任务分配给不同 agent 或按同样顺序由主 agent 执行。

需求分析 锁定范围、安全边界、节点类型、脚本 API 和验收标准。 后端实现 实现脚本 runtime、graph 接入、超时、输出、状态和资源清理。 前端和 Monaco 实现脚本编辑器、动态补全、hover、diagnostics 和节点内容区。 工作区 / MCP / Demo 覆盖保存恢复、MCP graph 工具、脚本 demo 和可复现示例。 审查 重点检查越权、资源泄漏、运行态一致性和拓扑语义破坏。 验证 执行 Go、Vitest、类型检查、构建、demo 自动运行和压力测试。 Agent 任务分配流程图 展示接口冻结、并行开发、集成、审查、验证和每个 agent 内部循环。
