Phase 1

拓扑内脚本能力

第一阶段的目标是让脚本成为普通拓扑节点。脚本只通过节点输入和输出端口处理数据，不允许直接打开串口或访问系统资源。

Generator定时生成 bytes
Transform处理并输出 bytes
Receiver / Analyzer显示或解析结果

目标

建立脚本功能的最小闭环，让用户不用离开拓扑图就能编写自动发送、数据转换和协议分析逻辑。

脚本处理器

`serial.script.transform` 接收 bytes，运行脚本，输出处理后的 bytes。

脚本发生器

`serial.script.generator` 按启动或定时器执行脚本，输出模拟数据。

脚本分析器

`serial.script.analyzer` 只消费数据，把字段、错误和帧记录写入内容区域。

实现

复用现有 graph runtime 的 bytes 路由，新增脚本 runtime wrapper。现有协议脚本引擎只作为基础能力参考，不直接承担拓扑节点生命周期。

节点配置

  • `script`：脚本源码。
  • `timeoutMs`：默认 50ms。
  • `encoding`：默认 utf-8。
  • `maxOutputBytes`：默认 65536。
  • `autoRun` 和 `intervalMs`：仅 generator 使用。

安全边界

  • 只暴露 PortWeave API。
  • 不开放文件、网络、进程和环境变量。
  • 脚本错误只影响当前节点。
  • 图停止必须终止定时器和执行任务。

运行流程

所有数据流都经过连接线，因此计数、监控、buffer、demo 和测试都能观察到脚本行为。

Step 1

图启动

为每个脚本节点创建独立 runtime state。

Step 2

收包

transform/analyzer 从输入端口收到 bytes。

Step 3

执行

脚本在超时和输出限制内执行。

Step 4

输出

transform/generator 通过 output API emit。

Step 5

记录

fields/errors 写入 frame history。

脚本 API

第一阶段 API 只覆盖输入、输出、字段标注、错误记录、状态和常用工具。

input.bytes()
input.hex()
input.text("utf-8")

output.bytes([0x01, 0x02])
output.hex("01 02")
output.text("OK\r\n", "utf-8")
drop()

field("unitId", 1, "Unit 1")
error("crc mismatch")

state.get("seq")
state.set("seq", 12)

crc16(input.bytes())
sum8(input.bytes())
now()

验收

功能通过自动化测试和 demo 验证，不依赖人工观察。

必须通过

  • transform 能把输入改写后传给 receiver。
  • drop 后下游不收到数据。
  • generator 启动后循环输出。
  • analyzer 能产生 frame 字段和错误。
  • 停止图后计数不再增长。

不能出现

  • 脚本能访问系统资源。
  • 脚本异常导致整个图崩溃。
  • 停止后 goroutine 或定时器残留。
  • 保存恢复后脚本源码丢失。
