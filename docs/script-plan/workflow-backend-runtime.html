Workflow 2

后端脚本 Runtime 实现阶段

目标是让脚本节点能在 graph runtime 中安全执行、路由数据、记录结果，并在停止时完整清理。

receive节点收到 bytes
script runtime超时执行
emit/frame输出或记录

实现目标

后端 agent 只负责 runtime、执行安全和 Go 测试，不处理前端编辑器。

脚本执行

每个节点独立状态，支持超时、输出限制和错误记录。

数据路由

transform/generator 输出进入 graph emit，analyzer 写 frame。

生命周期

图停止取消定时器和执行任务，释放 endpoint 资源。

步骤

先建立最小 transform 闭环，再扩展 analyzer、generator 和 endpoint。

Step 1

失败测试

写 transform 输入输出测试。

Step 2

Provider

后端 catalog 增加脚本节点。

Step 3

Wrapper

封装 goja 脚本 runtime。

Step 4

Receive

接入 graph receive 分支。

Step 5

Generator

接入定时执行和停止清理。

Step 6

Endpoint

第二阶段实现资源绑定。

测试重点

后端测试必须覆盖错误路径和停止路径。

行为测试

  • transform 输出改写数据。
  • drop 不发送下游。
  • analyzer 生成字段和错误 frame。
  • generator 最小 10ms 间隔输出。

安全测试

  • 死循环脚本超时。
  • 输出和状态大小超限。
  • 停止后计数不再增长。
  • endpoint 只能写绑定资源。
