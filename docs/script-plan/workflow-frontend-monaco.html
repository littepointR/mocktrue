Workflow 3

前端节点内容区和 Monaco 编辑器

目标是让脚本编辑体验接近 IDE：根据节点类型动态提示可用 API，并在用户写错安全边界时给出明确诊断。

Node Typetransform / generator / analyzer
API Model动态声明和补全
Monaco编辑、hover、诊断

实现目标

新增专用 ScriptEditor 组件，替换普通 textarea，并接入节点 config 保存。

编辑器

引入 Monaco，固定容器，可拖动高度，跟随主题。

动态补全

不同节点暴露不同 API、模板和参数片段。

诊断

禁止 API、节点不匹配 API 和风险调用即时提示。

Monaco 能力

补全和诊断由 PortWeave 脚本 API 模型生成，不从 runtime 反推。

能力内容示例
Completion函数、参数片段、模板`output.hex("01 02")`
Hover用途、参数、节点限制`drop()` 终止本次输出
Diagnostics禁用 API 和节点错配generator 中使用 `input.*` 报错
Theme跟随系统和应用主题亮色/暗色自动切换

实现流程

先让组件可测试，再接入真实 Monaco worker 和构建。

Step 1

依赖

添加 monaco-editor 和 Vite worker 配置。

Step 2

组件

实现 ScriptEditor props、events、主题。

Step 3

API 模型

按节点类型生成补全和声明。

Step 4

接入内容区

脚本字段写回节点 config。

Step 5

测试

Vitest mock Monaco，build 验证真实加载。

验收

编辑器不能只是可输入，必须真正提供动态提示和补全。

必须通过

  • transform 有 input/output 补全。
  • generator 没有 input 补全。
  • analyzer 使用 output 有诊断。
  • endpoint 才显示 serial API。
  • 主题切换后 Monaco 主题同步。

布局要求

  • 编辑器在固定容器内滚动。
  • 不导致父布局无限增长。
  • 切换节点后内容不丢失。
  • 运行中修改显示需重启提示。
