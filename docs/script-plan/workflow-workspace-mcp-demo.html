Workflow 4

工作区、MCP 和 Demo 集成

目标是让脚本节点成为 PortWeave 的普通能力：能保存恢复、被 MCP 创建和启动、在 demo 中完整展示，并且没有隐藏特殊逻辑。

Workspace保存脚本和布局
MCP列出和启动节点
Demo可复现自动运行

目标

脚本节点不能只在 UI 中可见，还要贯穿配置文件、自动化接口和示例工作区。

保存恢复

脚本源码、节点配置、布局、标签页和历史数据完整导入导出。

MCP 暴露

provider catalog、validate、start、status、query 都支持脚本节点。

Demo 可复现

用户可通过手动节点操作搭出与 demo 一样的内容。

实现步骤

先同步 provider 和 snapshot，再做 demo 和 MCP 验证。

Step 1

Snapshot

确认脚本 config 进入 graph workspace state。

Step 2

MCP catalog

同步脚本 provider、默认配置和规则。

Step 3

Validate

脚本端口、fan-out、资源冲突校验。

Step 4

Demo

新增 transform/analyzer demo。

Step 5

Round trip

导出后导入，内容完全一致。

Demo 设计

Demo 必须基于真实节点和真实运行时，不允许特殊处理。

Demo拓扑展示能力
script-transform-demogenerator -> transform -> monitor脚本改写数据、自动发送、监控帧显示
script-analyzer-demogenerator -> tap -> analyzer + monitor字段解析、分流、监控对照
script-endpoint-demovirtual port + endpoint buffer + monitor第二阶段受控资源访问

验收

集成目标以可重复自动化验证为准。

必须通过

  • 所有 demo 的 graph validate 通过。
  • 所有 demo 启动后计数增长。
  • 导出导入后脚本源码一致。
  • MCP 能 validate 和 start 脚本图。

不允许

  • demo 依赖隐藏运行逻辑。
  • 加载不同 demo 得到同一个拓扑。
  • MCP catalog 与 UI provider 不一致。
  • 导入导出遗漏脚本字段。
