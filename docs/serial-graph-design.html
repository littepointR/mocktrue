PortWeave Serial Graph Design

把串口功能变成模块节点，在连接工作台中拼接、运行和保存

连接界面不写死串口、Modbus 或 FECbus 逻辑。每个模块只暴露自己的节点、端口、配置面板和运行时工厂，图形化工作台负责拖拽、连线、校验、保存布局，统一运行时负责资源创建、数据路由和清理。

物理串口 独占打开真实端口，产生 rx/tx 数据流
Tap / Tee 把一份数据流分发给多个消费者
协议节点 Modbus RTU、FECbus 或自定义解析
显示与记录 接收窗口、日志、文件记录、统计
1 真实端口唯一 owner
N 并联消费者
0 停止后残留资源

总体结构

模块注册节点能力，连接工作台只处理通用图模型。运行时把图转换为已有 service 调用，因此用户通过图形界面拼出来的内容，和手动创建串口、桥接、监听、协议会话是同一套能力。

模块如何暴露能力

每个模块只注册自己能提供的节点。连接界面不理解协议细节，只读取 provider 的端口定义、校验函数和运行时工厂。 

interface SerialGraphNodeProvider {
  type: string
  title: string
  category: "port" | "virtual" | "bridge" | "monitor" | "protocol" | "tool"
  inputs: PortSpec[]
  outputs: PortSpec[]
  defaultConfig: unknown
  ConfigPanel: Component
  validateConfig(config: unknown): ValidationResult
  validateConnection?(from: PortSpec, to: PortSpec): ValidationResult
  createRuntime(config: unknown, context: GraphRuntimeContext): Promise<RuntimeNode>
}

节点端口模型

端口类型决定能不能连接。真实串口节点提供字节流，协议节点消费字节流并输出帧、寄存器或解析结果，显示节点消费结果。

串联与并联

串联是单一路径处理，并联必须通过 Tap/Tee 分发。这样真实端口只被一个 owner 打开，避免多个功能抢同一端口导致冲突。

串联处理

用于单向处理链路，例如读取数据、解析协议、展示结果。

物理串口/dev/ttyUSB0
Modbus RTU帧解析和寄存器映射
接收显示表格、HEX、ASCII

并联监听

用于同一份数据同时进入显示、日志和协议解析。

物理串口唯一资源 owner
Tap / Tee分发原始字节流
接收显示用户观察
文件记录长期留存
FECbus 解析标注帧字段

节点目录

第一版只需要覆盖现有功能，后续新增协议时只注册新 provider，不改连接工作台主体。

模块 暴露节点 输入 输出 运行时动作
串口 物理串口发送器接收器 bytes / config bytes / status 调用 openPort、send、close、restore counters
虚拟串口 虚拟串口 config port / bytes 创建一个虚拟串口并登记清理任务
桥接 桥接TapTee port / bytes bytes / mirror 建立双向转发，支持并联消费者
监听 串口监控帧表 port / bytes frames / counters 启动监听、查询帧、清空记录

FECbus 主控节点电气控制装置字段标注 bytes / request annotated frames / units 打开会话、发送帧、多从站、查询标注

运行时规则

图能不能运行，不由界面猜测，而由运行时统一校验和规划。失败时必须可解释、可回滚、可清理。

资源所有权

  • 真实串口只能有一个 owner
  • 虚拟串口由运行计划创建
  • Tap/Tee 只消费 owner 输出的数据流

连接校验

  • bytes 只能接 bytes
  • frame 只能接协议显示或记录
  • 协议 Master/Slave 角色在节点创建时确定

生命周期

  • 启动按依赖顺序执行
  • 任一步失败则反向清理
  • 停止图释放所有串口、桥接和监听资源

保存格式

配置文件中保存图、节点配置、布局、运行快照和历史数据。导入后连接工作台应恢复到导出时的完整状态。

  }
}

落地顺序

先做能保存和展示的连接工作台，再接运行时。这样可以让用户先验证图形化表达是否符合直觉，再逐步打开真实资源编排能力。

Provider 接口

定义节点、端口、配置面板和运行时工厂。

连接工作台

用 Vue Flow 实现拖拽节点、连线、选择和保存布局。

静态校验

检查端口类型、角色、真实串口 owner 和环路。

运行计划

把图转换成现有 service 调用，并支持失败回滚。

协议节点

接入 Modbus、FECbus、自定义解析和记录节点。
