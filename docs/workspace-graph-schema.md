# Workspace and Graph Schema

PortWeave currently uses TypeScript-defined JSON snapshots for whole workspaces and single graph tabs. This page documents the existing model; it does not claim a separate JSON schema validator or migration engine exists.

Source-of-truth: `frontend/src/workspace/workspaceSnapshot.ts` and `frontend/src/serial/graph/serialGraph.ts`.

## File kinds

Two snapshot kind strings are current:

- `portweave.workspace.v1` — whole workspace snapshot.
- `portweave.graph.v1` — single graph tab snapshot.

These strings are stored in `workspaceKind` and `graphTabKind`.

## When each kind is used

Use `portweave.workspace.v1` when saving/restoring the broader app workspace state:

- Settings.
- Serial terminal/session UI state.
- Virtual/bridge lists in saved state.
- Buffers.
- Monitor/Modbus/FECbus workspace stores.
- Serial graph workspace state.
- UI workspace/editor layout state.

Use `portweave.graph.v1` when exporting/importing one graph tab:

- Serial settings relevant to the graph.
- One `SerialGraphDocument`.
- A runtime snapshot containing node buffers and node frames.

## Workspace snapshot contents

`WorkspaceSnapshot` contains:

```text
kind: portweave.workspace.v1
settings.serial
serial.activePortId
serial.handles
serial.virtualPorts
serial.bridges
serial.buffers
serial.monitors
serial.modbus
serial.fecbus
serial.graph
serial.workspace
```

Important boundary: snapshot data is not the same thing as live OS handles. Demo snapshots intentionally avoid preloading open serial handles, runtime buffers, monitor sessions, protocol sessions, virtual ports, or bridge resources.

## Graph tab snapshot contents

`GraphTabSnapshot` contains:

```text
kind: portweave.graph.v1
settings.serial
graph: SerialGraphDocument
runtime.nodeBuffers
runtime.nodeFrames
```

`runtime.nodeBuffers` and `runtime.nodeFrames` are captured evidence, not a live running graph.

## SerialGraphDocument

A graph document contains:

```text
id
name
nodes
edges
selectedNodeId
selectedEdgeId
nodeTabs
activeNodeTabId
```

Each node contains:

```text
id
type
name?
position
config
status?
error?
```

Each edge contains:

```text
id
source
sourceHandle
target
targetHandle
```

`source` and `target` reference node IDs. Handle IDs must exist in the current provider catalog.

## Runtime snapshot boundaries

Persisted snapshots may include buffers/frames, but they do not persist:

- Running goroutines.
- Open serial/TCP file descriptors.
- Active reconnect loops.
- OS-level virtual ports as live resources.
- External device state.

After restore/import, start graph runtime again to recreate live resources.

## Import/restore behavior

Current helper functions clone graph documents and sanitize stale config fields:

- Monitor config drops legacy `mode`.
- Bridge config drops ignored `baudRate`.
- Modbus slave config drops ignored `addressMode`.
- Remote serial config drops `mode`.

Graph documents preserve node IDs and edge IDs within the document. Whole-workspace restore may handle live serial handles differently because live handles are runtime resources, not stable saved graph identity.

## Demo workspace relationship

Demo workspaces are generated by `frontend/src/workspace/demoWorkspaces.ts` and return `WorkspaceSnapshot` objects. They are read-only examples in UI terms, but they still use the same graph document/state shapes.

Demo IDs such as `remote-serial-demo` are catalog IDs, not graph document IDs. Each load produces fresh graph/resource IDs.

## MCP template relationship

MCP templates are backend graph config templates returned by `serial_graph_demo_template`. They provide nodes and edges for validation/start. They are not full frontend workspace snapshots and do not synthesize frontend operation-log state.

## Compatibility expectations

Current docs describe version `v1` kinds. If fields change, prefer additive compatibility and document migrations before relying on changed semantics. Do not assume a schema validator exists until code adds one.

## Minimal workspace excerpt

```json
{
  "kind": "portweave.workspace.v1",
  "settings": { "serial": {} },
  "serial": {
    "activePortId": null,
    "handles": [],
    "virtualPorts": [],
    "bridges": [],
    "buffers": {},
    "graph": {
      "graphs": [
        {
          "id": "graph-1",
          "name": "拓扑图 1",
          "nodes": [],
          "edges": [],
          "selectedNodeId": null,
          "selectedEdgeId": null,
          "nodeTabs": [],
          "activeNodeTabId": null
        }
      ],
      "activeGraphId": "graph-1"
    }
  }
}
```

## Minimal graph tab excerpt

```json
{
  "kind": "portweave.graph.v1",
  "settings": { "serial": {} },
  "graph": {
    "id": "graph-demo",
    "name": "Example graph",
    "nodes": [],
    "edges": [],
    "selectedNodeId": null,
    "selectedEdgeId": null,
    "nodeTabs": [],
    "activeNodeTabId": null
  },
  "runtime": {
    "nodeBuffers": {},
    "nodeFrames": {}
  }
}
```
